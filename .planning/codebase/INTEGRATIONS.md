# External Integrations

**Analysis Date:** 2026-07-19

## APIs & External Services

**AI Model Providers:**
- **Google Gemini** - Primary AI model provider for prompt execution and proposal generation
  - SDK/Client: `google-genai` (1.0+)
  - Config: `engine/ai/adapters/gemini.py`
  - Auth: Environment variable `ATLAS_GEMINI_API_KEY`
  - Model config: `ATLAS_GEMINI_MODEL` (e.g., `gemini-2.0-flash`)
  - Usage: Core proposal and review generation across all engineering subsystems

- **Anthropic (Claude)** - Alternative AI provider via HTTP
  - SDK/Client: None (uses `urllib` standard library HTTP requests)
  - Config: `engine/ai/adapters/anthropic.py`
  - Auth: Environment variable `ATLAS_AI_API_KEY`
  - Model config: `ATLAS_AI_MODEL`
  - Endpoint: `https://api.anthropic.com/v1/messages` (configurable via `ATLAS_AI_ENDPOINT`)
  - Usage: Optional fallback AI provider

- **OpenAI-Compatible** - Generic OpenAI protocol adapter for local or cloud endpoints
  - SDK/Client: None (uses `urllib` standard library HTTP requests)
  - Config: `engine/ai/adapters/openai_compatible.py`
  - Auth: Environment variable `ATLAS_AI_API_KEY`
  - Model config: `ATLAS_AI_MODEL`
  - Endpoint: `ATLAS_AI_ENDPOINT`
  - Usage: Local models or third-party OpenAI-compatible services

- **Ollama** - Local AI model runner
  - SDK/Client: None (uses `urllib` standard library HTTP requests)
  - Config: `engine/ai/adapters/ollama.py`
  - Endpoint: Typically `http://localhost:11434`
  - Usage: Offline-capable development and testing

## Data Storage

**Databases:**
- **None** - No traditional database integration. ATLAS uses filesystem-based storage.

**File Storage:**
- **Local filesystem only** - All project data persists to `.atlas/` workspace directory
  - Location: Configured via `ATLAS_WORKSPACE_ROOT` setting (default: `./workspace`)
  - Structure: Hierarchical JSON and markdown files managed by subsystems
  - Subsystems: Project, Research, Planning, Architecture, Knowledge, Evaluation
  - AI state: Pending proposals stored in `engine/ai/fs_repository.py` (filesystem-backed)

**Caching:**
- None configured (stateless request-response pattern for AI providers)

## Authentication & Identity

**Auth Provider:**
- Custom API-key-based (environment variable injection)

  - Example: Gemini uses `ATLAS_GEMINI_API_KEY` passed to Google SDK
  - Example: Anthropic uses `ATLAS_AI_API_KEY` passed in `x-api-key` header

**Identity Management:**

## HTTP Communication

**Transport Layer:**
- Python standard library `urllib.request.urlopen()` used in `engine/ai/adapters/_http.py`
- No third-party HTTP client (requests, httpx, aiohttp)
- Timeout: 60 seconds for AI provider requests
- Protocol: HTTPS to all external AI provider endpoints
- Error handling: HTTP errors, URL errors, JSON decode errors normalized to `AIProviderException`

## Provider Architecture

**Adapter Pattern:**
- All external AI providers implement `engine/ai/provider.py:AIProvider` abstract interface
- Protocol-agnostic request: `engine/domain/ai.AIRequest` (system prompt, user prompt, parameters)
- Protocol-agnostic response: `engine/domain/ai.AIResponse` (content, usage metrics)
- Provider-specific imports and SDK usage scoped to individual adapter modules (dependency inversion)
- Factory resolution: `engine/ai/factory.py` instantiates correct adapter based on `ATLAS_AI_PROTOCOL` config
